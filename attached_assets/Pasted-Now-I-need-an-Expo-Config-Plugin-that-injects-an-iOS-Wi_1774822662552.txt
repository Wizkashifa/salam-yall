Now I need an Expo Config Plugin that injects an iOS Widget Extension into the Xcode project at prebuild time.

Create a file called plugins/withPrayerTimesWidget.js that:

1. Is a valid Expo Config Plugin using withXcodeProject and withInfoPlist
2. At prebuild, it:
   a. Creates a new Widget Extension target called "PrayerTimesWidget" with bundle ID "app.ummahconnect.PrayerTimesWidget"
   b. Adds the "group.app.ummahconnect" App Group entitlement to BOTH the main app target and the widget target
   c. Creates the widget extension directory at ios/PrayerTimesWidget/ with these Swift files:
      - PrayerTimesWidget.swift (the main widget struct)
      - PrayerTimesEntry.swift (the timeline entry model)
      - PrayerTimesProvider.swift (the TimelineProvider that reads from UserDefaults suite "group.app.ummahconnect")
      - PrayerTimesWidgetViews.swift (the SwiftUI views for small, medium, and large widget families)
   d. Sets the widget's deployment target to iOS 17.0
   e. Adds the widget target to the Xcode project with proper build settings

3. Register this plugin in app.json/app.config.js under "plugins"

The widget should:
- Support .systemSmall (next prayer countdown), .systemMedium (all 5 prayers with completion status and interactivity),
- Read prayer data from UserDefaults(suiteName: "group.app.ummahconnect") key "prayerData"
- Display prayer name, athan time, iqama time, and use the same functionality as our Home Screen/prayer tracker for completed, missed, excused, etc prayers
- Use the same color scheme as our cards on the Home Screen within our app and support the same dark/light mode options. 
- Support the interactive widget button (iOS 17+) that toggles prayer completion status directly from the widget using AppIntent
- After toggling, write the updated status back to the shared UserDefaults and reload the timeline

For the AppIntent-based interactive button:
- Create a TogglePrayerIntent that conforms to AppIntent
- It should accept the prayer name as a parameter
- On perform(), it reads current data from shared UserDefaults, toggles the status (null → completed, completed → null), writes back, and reloads the widget timeline
- Wire this intent to a Button in the widget view for each prayer row
